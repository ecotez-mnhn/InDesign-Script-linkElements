    ▲ PRÉSENTATION GÉNÉRALE

    Version du 18 avril 2023.

    Màj : ajout de la fonctionnalité de gestion des références.

    Avertissement :

    • cette mise à jour majeure du script a apporté quelques changements dans l'interface du programme. Les fonctionnalités restent les mêmes, mais le texte des boutons a parfois changé ;
    • pour surligner les appels, il faut maintenant d'abord sélectionner un type d'appel (tous les appels ne sont donc pas surlignés en une seule fois) ;
    • les appels aux figures/tableaux/annexes et les appels aux références ne peuvent pas être liées en une seule fois.

    Objet :

    Ce programme permet :

    • Figures, Tableaux Annexes :

      • de lier les appels aux Figures, Tableaux et Appendices aux légendes correspondantes en une seule passe (appels complexes inclus) ;
      • de tester l'absence ou la présence des appels dans le texte, et de les surligner ;
      • d'insérer ou de supprimer des appels (en cas d'ajout ou de suppression de figure).

    • Références bibliographiques :


      • De créer les ancres (références courtes) à partir de la bibliographie ; une fois créées, il est possible de corriger les ancres manuellement ;
      • De détecter les appels à la bibliographie dans l'article (appels complexes et auteurs de taxons compris, tous blocs confondus) ; une fois les appels relevés, il est possible de les compléter ou de les corriger manuellement ;
      • De lier les appels aux entrées de bibliographies (appels complexes compris) ; une fois les références liées, il est possible de lier les références restantes manuellement ou de relancer le script une fois les appels/ancres corrigées ou complétées.

    Installation :

    Pour installer le programme, les fichiers suivants doivent être copiés dans le dossier « script » d'InDesign :

    • linkElements.jsx : Le programme de base ;

    • linkElements.conf : Le fichier contenant les options de configurations ;

    • linkElements_UI.jsx : L'interface graphique du programme ;

    • linkElements-help.html  : Ce fichier d'aide, au format HTML ;

    • linkElements-help.html_files  : Le dossier contenant les images du fichier d'aide.

    Lancement du programme :

    Pour lancer le programme :

    • ouvrir le fichier InDesign d'un article ;

    • double-cliquer sur le fichier  « linkElements.jsx » ;

    Le programme s'ouvre alors et lance automatiquement une analyse de l'article :

    Une fois l’analyse terminée, le résultat s’affiche dans le bloc INFORMATIONS du programme :

    Attention : Les appels aux Figures, Tableaux et Annexes détectés pendant l’analyse correspondent aux appels non liés précédemment.

    Interface :

    L’interface du programme est uniquement en anglais. Les couleurs et les styles de caractères utilisés par le programme, si ils n’existent pas, sont automatiquement ajoutés à la palette des couleurs et/ou des styles.

    • Le bouton « Exit » permet de quitter le script (tout comme la croix rouge).

    • Le bouton « Help » renvoie vers cette page ; en plus des explications données ici, un survol des différents boutons de l’interface permet d’afficher une infobulle qui vous renseignera sur leur fonction.

    Voici les différents blocs permettant d’interagir avec l’article :

    Pour activer les fonctionnalités proposées par le programme, il faut d’abord sélectionner un ou plusieurs type(s) d’éléments (« Figures », « Tables » ou « Appendices ») :

    Remarque : certaines fonctionnalités ne peuvent être utilisées que sur un seul type d’élément à la fois (Figures OU Tables OU Appendices), d’autres sur plusieurs types d’éléments (Figures ET/OU Tables ET/OU Appendices) (voir capture précédente).

    Avantages :

    Par rapport à la liaison entre appels et légendes via les références croisées :

    • la liaison des appels aux éléments se fait en une seule passe, voire en une seule passe pour tous les types d’éléments ;

    • les appels complexes sont gérés (ex. : Figs 3 ; 4A-C, G ; 5-7 ; 12) ;

    • les appels ne sont pas modifiés lors de la renumérotation (les « Figs » ne deviennent pas des « Fig. » lors de la mise à jour des appels) ;

    • aucune limite de taille n’est imposée au niveau de l’article et/ou du nombre d’éléments à traiter (les liens hypertexte et les ancres ne ralentissent pas le fichier comme le font les références croisées) ;

    • des outils vous permettent de placer vos figures ou de vérifier les appels simplement, sans toucher au texte.

    ▲ Bloc LIAISON :

    Choix du langage

    • Ces  deux boutons radio (« French » et « English »)  correspondent à la  langue de l’article (par défaut « English ») ;

    • Les  éléments détectés et liés le  seront  uniquement dans la langue sélectionnée ;

    Important :  lors d’un clic sur l’un des boutons « French » ou « English », le programme met à jour les contenus et appels correspondant à la langue dans le bloc INFORMATIONS (cela peut prendre quelques secondes/minutes en fonction de la taille de l’article).

    Remarque :  si l’article est bilingue, le programme de liaison devra être exécuté deux fois : une première fois pour détecter les appels dans une langue (ex. : « Table »), une seconde fois pour les appels dans l’autre langue (ex. : « Tableau »).

    Ajout de liens

    Avant d’utiliser ce bloc, il est conseillé de vérifier la présence des appels dans le texte en utilisant le bloc « ASSISTANCE ».

    Le bloc « Liaison » est dédié à l’ajout de liens hypertextes sur les appels détectés dans le texte vers le ou les type(s) d’élément(s) sélectionné(s).

    Cette partie peut être lancée sur plusieurs éléments à la fois (par ex. « Figures » ET « Tables »).

    Pour lier les appels à un type d’élément :

    • sélectionnez un ou plusieurs type(s) d’élément(s) (Figures, Tables ou Appendices) ;

    • cliquez sur « Launch linking ».

    Le programme effectue alors les actions suivantes :

    • Ajout des ancres aux légendes des éléments sélectionnés :

      • les ancres sont créées uniquement si elles n’existent pas encore (voir ici pour nettoyer les ancres inutilisées) ;

      • le programme vous fait valider la création de chaque ancre :

        • la légende détectée est affichée à l’écran ;

        • une boîte de dialogue est ouverte, avec le numéro d’ancre proposé (Fig. 1, Fig. 2, etc.) ;

        • si le numéro est correct, appuyer sur « OK » ;

        • s’il est incorrect, le corriger (taper uniquement le nombre, sans « Fig. » ou « Table » ou « Appendix » devant) et cliquer sur « OK ».

    • Recherche des appels dans le texte et ajout de liens hypertextes vers les ancres précédemment créées :

      • les appels liés sont passés en style « Appel de figure » et apparaissent en rose (si le style n’existe pas, il est automatiquement créé) ;

      • les appels en échec (non liés) ne sont pas stylés et apparaissent surlignés en bleu :

      • à la fin, le programme affiche le nombre de liens créés et le nombre d’appels en erreur :

    Remarques  : le programme lie les appels complexes (ex. : Figs 1A, C; 3B; 5 ). Les appels présents uniquement à l’intérieur d’une plage (ex. : la Fig. 14 à l’intérieur de l’appel aux Figs 13-15) ne sont pas liés (pour lier ces appels, utiliser d’abord la fonction « Expand ranges »).

    Suppression des liens

    Si vous avez besoin de supprimer les liens créés, il faut utiliser la fonction « Delete links » et le bouton correspondant. Pour cela :

    • sélectionnez un ou plusieurs type(s) d’éléments (Figures, Tables ou Appendices) ;

    • cliquez sur le bouton « Delete links » :

    Suppression des ancres inutilisées

    Si des ancres surnuméraires ont été créées (repérables de multiples signes « : » rouges au début des légendes ou des entrées de bibliographie), il est possible de les supprimer :

      Pour cela :

    • lancez le script « text_anchor.jsx » ;

    • cliquez sur le bouton « select unused » ;

    • cliquez  sur le bouton « delete selected ».

    ▲ Bloc INFORMATIONS :

    Figures, Tableaux et Annexes

    Ce bloc affiche les informations sur les Figures, Tableaux et Annexes détectés dans l’article et sur leurs appels détectés dans le texte :

    • Les éléments (Figures, Tables ou Appendices) sont détectés en fonction du style de paragraphe de leurs légendes (style de paragraphe recherché : « Légende figure ») ;

    • Les appels aux Figures, Tables et Appendices sont détectés en fonction de la langue sélectionnée (par défaut, « English ») ;

    • Les appels complexes (ex. : Figs 1A, C; 3B; 5) sont également pris en compte, mais pas les plages (un appel aux Figs 13-20 ne compte que pour 2 appels dans le total affiché).

    Références bibliographiques : voir la partie "Références bibliographiques".

    ▲ Bloc ASSISTANCE :

    Ce bloc met à votre disposition une série d’outils vous permettant de :

    • surligner  automatiquement les appels aux éléments dans le texte  (Spotlight  calls) ;

    • vérifier  les appels aux éléments présents dans le texte  (Check calls) ;

    • étendre  et regrouper les plages d’appels (Ranges) ;

    • supprimer une série d’appels dans le texte (quand un ou plusieurs éléments ont été  supprimés) et décaler les appels suivants (Delete  calls) ;

    • insérer un appel dans le texte (quand un ou plusieurs éléments ont été insérés), et décaler les appels suivants (Shit calls).

    Les modifications appliquées par le programme le sont en deux temps : les suppressions apparaissent d’abord barrées de rouge, et les ajouts soulignés de vert ; un clic sur le bouton de validation (coche) permet ensuite d’appliquer les corrections définitivement. Il est toujours possible de revenir en arrière en utilisant les boutons « Undo » et « Redo ».

    Pour que les boutons s’activent, il faut d’abord sélectionner un ou plusieurs type(s) d’élément(s) (Figures, Tables ou Appendices OU Références).

    Surligner les appels aux éléments  (Spotlight calls)

    Cette fonction nécessite de sélectionner d’abord un type d’élément, net s’applique à tous les appels correspondants dans le texte :

    • Cliquez sur « ON » pour « allumer » les appels dans le texte ;

    • Cliquez sur « OFF » pour les éteindre.

    Étendre ou regrouper des plages d’appels (Ranges)

    Le programme permet d’étendre ou de regrouper les plages d’appel à un type d’élément (un seul type d’élément doit donc être sélectionné) :

    Remarque : comme pour les autres fonctionnalités du programme, l’extension ou le regroupement de plages ne fonctionne que sur les appels non encore liés (et donc, non stylés).

    Supprimer une série d’appels  (Delete calls)

    Le programme permet de supprimer une série continue d’appels (par ex. : Fig. 1 ou Figs 3-5), dans le cas où un ou plusieurs éléments auraient été supprimés de l’article :

    • Les appels aux éléments qui suivent l’élément supprimé sont renumérotés (ex. : si on supprime la Fig. 3 de l’article, le programme renuméroté les appels suivants, à partir de l’appel suivant, Fig. 4, en fonction du nombre de Figures supprimées [une seule dans notre cas] ; la Fig. 4 devient ainsi la Fig. 3, la Fig. 5 devient la Fig. 4, etc.) ;

    • La suppression se fait en deux temps :

      • les appels supprimés sont barrés de rouge et les appels décalés sont soulignés en vert lors du premier passage ;

      • si la suppression est validée, un clic sur le bouton « Coche » applique la transformation ;

      • les appels cités dans les plages sont pris en compte ;

      • il est possible de revenir en arrière avec les boutons « Undo » et « Redo ».

    Dans la capture suivante, on supprime les appels à l’Annexe 3 (appels en anglais) ; les appels suivants sont renumérotés :

    Dans la capture suivante, on supprime les appels à aux Tableaux 11 et 12 (appels en anglais) ; les appels suivants sont renumérotés :

    • Pendant l’exécution du script, le programme demande si on veut renuméroter les appels au-delà du nombre d’éléments trouvés dans l’article (par ex., si l’article contient 15 Figures, la renumérotation va s’arrêter à 15, mais on peut « forcer » la renumérotation des éléments suivants, pour renommer un appel à la Fig. 16 en Fig. 15, Fig. 17 en Fig. 16, etc.) ;

    • La renumérotation implique l’ajout d’espaces insécables entre le type d’élément et le numéro d’élément, et le remplacement de séries d’espaces en une seule espace non insécable partout dans l’article ;

    • Une fois la renumérotation terminée, il est important de lisser le texte de l’article « à la main » (gestion des pluriels [ex. : Figs en Fig.] et des éléments vides [ex. : (Figs )] , qui apparaissent en bleu à la fin de l’exécution de la fonction).

    Insérer une série d’appels  (Shift calls)

    De la même manière, le programme permet d’insérer un appel et de décaler les appels suivants. Par exemple :

    • si une figure, la « Fig. 3 » a été ajoutée, tous les appels aux Figures 3 à X doivent être décalés de 1 ; les appels à la Fig. 3 deviennent des appels la Fig. 4, les appels à la Fig. 4 deviennent des appels à la Fig. 5, etc.

    • si on ajoute plusieurs figures à la fois, par exemple les Figs 3, 4 et 5, les appels à partir de la Figure 3 doivent être décalés de 3 ; les appels à la Fig. 3 deviennent des appels à la Fig. 6, les appels à la Fig. 4 deviennent des appels à la Fig. 7, etc.

    ▲ Gestion des références/authorships  :

    Le script gère les références bibliographiques. Il permet :

    • de vérifier et créer les ancres (références courtes) ;
    • de repérer les appels dans le texte (y compris les appels complexes) ;
    • de lier les appels et ancres ;
    • de lier les références bibliographiques (sans virgule) et les authorships (avec virgule) ;
    • de faire la liste des références manquantes et des références non appelées ;
    • de copier/coller le résultat de l'opération dans un mail.

    Remarque : les captures d'écran correspondent à la version 1.3 du script linkElements.jsx.

    Activer la gestion des références

    Pour activer la gestion des références, cocher la case "Références", en haut de la fenêtre du script :

    Une fois cochée, la partie "Call management" (auparavant appelée "Correction assistance") prend la forme suivante :

    Remarque : les appels aux Figures/Tableaux/Annexes et les appels aux Références ne peuvent pas être gérés en même temps.

    Étape 1 : Création des ancres

    Pour lancer la création des ancres, procédez de la manière suivante :

    • Cochez la case "References" (en haut de la fenêtre du script) ;
    • Cliquez sur le bouton "Create anchors" :
      • le script parcourt les références et génère les références courtes (ancres) ;
      • les références courtes sont listées dans la fenêtre de log du script, pour contrôle visuel ;
      • si des ancres en double sont trouvées (il manque les petites lettres a, b, c, etc.), le script affiche un message d'erreur et invite à corriger les références correspondantes (surlignées en rose) avant de continuer ; 
      • si aucune ancre en double n'est trouvée, l'exécution du script continue ;
      • le contrôle visuel des ancres est correct, la création des ancres peut être validée.

    Si le contrôle visuel des références courtes n'est pas correct, ou si des ancres en double sont détectées, il est possible de corriger les références (surlignées en rose) et de relancer ensuite la création des ancres.

    Correction des ancres :

    Une fois les ancres créées, il est possible (et même recommandé) de les contrôler/corriger avec le script txt_anchor.js.

    Les ancres mal formées sont souvent le reflet d'une entrée de bibliographique mal formatée (absence de "—" par exemple).

    Important :

    1°/ les ancres sont maintenant estampillées "REF_" et numérotées (pour préserver l'ordre des références) :

    Si les ancres doivent être modifiées, il est important de conserver les 9 premiers caractères inchangés ("REF_XXXX_").

    2°/ Si le création des ancres est relancée alors qu'elles existent déjà, les ancres déjà existantes (et non déplacées) ne seront pas recréées :

    3°/ D'une manière générale, si des références doivent être corrigées ou déplacées, les ancres doivent être supprimées (en utilisant le bouton "Del. anchors") et créées à nouveau.

    Étape 2 : Repérage des appels

    Une fois les ancres créées, il faut repérer les appels aux références dans le texte avant de lancer la liaison appels/références. Pour cela, cliquez sur le bouton "ON" de la partie "Call management" (le bouton radio "References" est coché).

    Le script tente alors de surligner les appels repérés dans le texte :

    Corriger les appels mal repérés

    Certains appels sont mal repérés par le script (typiquement, les noms composés sans tiret) ; il est maintenant possible de les corriger :

    Pour cela, sélectionnez une (et une seule) référence complète (dans l'ex. ci-dessus, sélectionnez "Royo y Gómez (1941" ;

    Cliquez sur le bouton "AddRef" ou lancez le script linkElementsAddCall.js (les 2 font exactement la même chose). L'ensemble des occurrences de la référence sélectionnée sont maintenant repérées dans le texte :

    De cette manière, il est possible de corriger les appels de références avant de lancer la liaison appels/références, et de minimiser ainsi le nombre d'appels en échec.

    Il est possible de mettre un raccourci sur le script "linkElementsAddCass.js" de manière à "tagguer" plus rapidement les appels aux références ratés par le script.

    Appels complexes

    Les appels complexes (du type Auteur 2009; 2010a; 2012b, c) sont directement gérés par le script.

    Notez que les appels aux références sont bien séparés les uns des autres lors du repérage :

    (les points-virgules et les virgules ne sont pas surlignés en mauve lors du repérage par le script ; si vous complétez manuellement les appels, faites bien attention à respecter cette règle).

     Étape 3 : lier appels et références

    Une fois les ancres créées et les appels relevés, il est possible de lancer la liaison entre les deux ; une fois l'opération effectuée, le script affiche un rapport 

    – les appels en échec sont surlignés en bleu foncé ; ils correspondent à des références manquantes ou à de mauvais appels ;

    – les appels corrects, mais sur lesquels la création de lien a échoué, sont surlignés en bleu clair ; il faut supprimer manuellement le lien qui existe sur ces éléments avant de relancer la liaison appels/références ; 

    – les appels correctement liés sont stylés dans le style "Appel de référence lié", et apparaissent colorés en rouge dans le document :

    Appels en échec

    Les appels en échec (surlignés en bleu foncé dans le texte) correspondent soit :

    • à des références manquantes ;
    • à des ancres mal formées (solution : corriger l'ancre) ;
    • à des appels mal formés (solution : corriger l'appel, éventuellement après confirmation de l'auteur).

    (Dans l'exemple ci-dessus, l'appel contient un "y" minuscule et la référence courte un "Y" majuscule).

    Il est ensuite possible :

    • de corriger les appels manuellement, en ajoutant un lien hypertexte vers l'ancre correspondante ;
    • de relancer la liaison appels/références une fois les appels et/ou les ancres corrigés.

    Si la liaison appels/références est relancée, les appels déjà liés sont ignorés, mais utilisés pour la gestion des appels complexes :

    Résultats

    Référence manquantes

    Les références manquantes (appelées dans le texte mais non présentes en bibliographie) apparaissent, après contrôle, en bleu foncé dans le document (voir capture ci-dessus).

    Le script génère aussi un rapport à la fin de la liaison, qui s'affiche à l'écran de l'utilisateur et est enregistré dans la fenêtre de log du script. Il est possible de copier/coller la liste des références manquantes (pour chaque appel en échec, le numéro de page est indiqué) pour l'envoyer à l'auteur :

    Le contenu de la fenêtre (qu'il est possible de faire défiler) est également enregistré dans la fenêtre de log du programme.

    Références non appelées

    Le script surligne également les références non appelées ; à la fin de l'exécution de la liaison, celles-ci apparaissent en bleu foncé dans la bibliographie.

    Remarque : avant de supprimer les références surlignées pendant la liaison appels/références, il faut vérifier que les références ne correspondent pas à des auteurs de taxon (voir ci-dessous). 

    Vérifier et lier les auteurs de taxons

    Le script permet également de vérifier les auteurs de taxons. Pour cela, il faut cocher le bouton radio "Authorships" au lieu de "References":

    Le procédé est ensuite le même qu'avec la liaison des appels aux références :

    – détecter les auteurs avec le bouton "ON" ;

    – lancer la liaison avec le bouton "Launch linking".

    Pointage des références non appelées en prenant en compte les authorships

    Pour déterminer quelles références bibliographiques ne sont pas appelées dans l'article, procéder comme suit :

    •  Lier les appels aux références :
      • Le script "allume" en bleu les références non appelées (1er passage) ;
    • Lier les auteurs de taxons aux références :
      • Le script "éteint" les références allumées correspondant à des authorships.

    À ce stade, les références bleu foncé ne correspondent ni à un appel classique (sans virgule), ni à un auteur de taxon (avec virgule).
